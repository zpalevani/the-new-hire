# Module 0 · Paperwork 📋
### ⏱️ ~20 minutes · Cost: $0

Every onboarding starts with paperwork. Yours is mercifully short.

## What you need

- A computer running **macOS, Windows, or Linux**
- **Google Chrome** installed
- A **personal Gmail account** (work/school accounts can hit restrictions — use personal)
- That's it. **No billing. No credit card. No terminal.**

## Step 1 — Install Antigravity

Antigravity is Google's command center for AI agents. Think of it as the office your new hire works in — you can see everything they do from your desk.

1. Go to the [Antigravity download page](https://antigravity.google/download) and download **Antigravity** (the standalone app, white icon) for your operating system.
2. Run the installer. Defaults are fine.
3. Launch it and click through the setup: choose **fresh start**, pick a theme you like.

> 🖥️ **The IDE question.** The download page also offers "Antigravity IDE" — a programmer's editor with an agent inside. You don't need it. The standalone app was built precisely so that agents aren't chained to a code editor. Skip it.

## Step 2 — Sign in

Sign in with your **personal Gmail** when prompted. Antigravity is in free preview for personal accounts and includes free quota on premium models. Let it create a new Chrome profile if it asks — that's how your agent gets its own supervised browser later.

Accept the terms, make your own call on the data-sharing opt-in, and land on the main screen.

## Step 3 — Create the workspace

On your Desktop (or wherever you keep work), create a normal folder called:

```
triage-desk
```

Right-click → New Folder. That's the extent of your technical setup. This folder is your new hire's desk — everything they build lives here, where you can see it.

## ✅ Checkpoint

- [ ] Antigravity opens and you're signed in
- [ ] A folder named `triage-desk` exists on your machine
- [ ] You have not opened a terminal (and you won't)

## 🎩 What just happened

Nothing fancy — but notice what you *didn't* do. No SDKs, no package managers, no environment variables. The entire developer-tooling layer is your agent's problem now. That division of labor is the whole workshop.
